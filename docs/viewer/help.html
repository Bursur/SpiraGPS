Route Editor - Help

This is a quick guide to the different parts of the route editor. It contains a list of the different components that are supported in the editor and how they work together.

The visuals of the route editor have been designed so that they look as similar as possible to what the reader will see when they view the route in the viewer. To make navigation easier panels can also be collapsed to free up some space.

Panels

There are a number of different panels that can be used to build up a run. These mostly behave in the same way, but they provide an easy way for people to identify what to expect at that point in the run.

All panels have a few different options available to them. You can state what conditions need to be met for the panel to be displayed, you can also choose if the panel is minimised or maximised when the route is first loaded.

You add a panel by clicking on the "Add Panel" button that appears at the end of the current entries in the route. The type of panel is set by selecting the type of the panel il in the dropdown that appears.

To edit a panel, you just need to click on it. You can then change the conditions, position and delete the panel, you can also duplicate it.

These are the different kinds of panel available:

Battle Panel - 123,456 HP

  • This is a battle panel. It contains information about how to fight this particular boss.
  • It shows the name of the boss and (optionally) it's health and action count.
  • The "steps" in the panel are added as any entry type.
  • They can be added and removed via the edit panel.
ENCOUNTERS

  • This is an encounters panel. It contains information about what to do in the different random encounters in this area.
  • The instructions in this panel are added as any entity type.
  • The can be added and removed via the edit panel.

This is intended for information that isn't essential to the run, but will help beginners with some of the trickier aspects of the route.

The information in these panels is displayed as a list of all available component types.

Entries

Entries are the different components that can be added to the guide. They are the building blocks of the run, and there are a few different types that can be chosen from.

To add an entry to the run click on the "Add Entry" button at the bottom of the page and select the type of entry from the drop down list.

Entries can have conditions applied to them. It is possible to only show them in specific circumstances.

To edit an entry, you just need to click on it. You can then change the conditions, position and delete the entry, you can also duplicate it.

These are the available entries:

Info - This is just simple line of text.

Image - Embed an image into the run. You can set it's size by providing a percentage of the screen width that it should fill using the slider provided. If you set a URL the image will need to be stored somewhere, whereas if you choose a file it will be embedded into the notes themselves and won't need to be hosted elsewhere.

Conditions

Conditions are used to show and hide different parts of the guide depending on what options have been selected from the list that is anchored to the top of the screen in the route viewer.

You can add a condition by clicking on the "Update Conditions" button at the bottom of the screen. This will open a dialog where you can set the name of the condition and it's default state as well as all available states for that condition. Once a condition has been created each entry and panel can then be linked to it and will display based on the state selected when setting their condition.

Each entry or panel can have multiple logical AND conditions allowing for greater control over what is impacted by each condition.

You should use conditions to control what should show, rather than what shouldn't. This may mean having some duplicated items, but it makes editing the document a lot easier as you knowexactly what will show and when.

Keywords

Keywords are automagically highlighted by being made bold. You can add as many keywords you want to your guide.

There are a number of keywords that already exist within the SpiraGPS system. These are words that are common across all routes and often have more formatting associated with them than simply making them appear bold.

  • Tidus, Yuna, Auron, Wakka, Kimahri, Lulu, Rikku
  • Valefor, Ifrit, Ixion, Shiva, Bahamut, Anima
  • Seymour, Jassu, Keepa, Balgerda, Abus
  • Enemy, Any, All and BUG!

Contents

The contents list is generated automagically by the route viewer based on the chapter titles. You shouldn't need to worry about setting these, just make sure your chapter titles make sense for the reader.


You can use the content list at the edge of the editor screen to jump between chapters quickly.

Saving/Loading

Routes can be saved and loaded using the Load and Save buttons at the bottom of the screen.

ROUTES ARE NOT AUTO SAVED!

It is your responsibility to remember to save your work! However you will be shown a confirmation when you try to close or navigate away from the editor.
